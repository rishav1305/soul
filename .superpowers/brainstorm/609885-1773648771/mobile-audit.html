Soul v2 — Mobile Chat Audit

Current issues found on screens <640px. Which problems bother you the most?

1

Toolbar Overflow

7 toolbar items (Product, Mode, Model, Think, Attach, Camera, Send) crammed into one row. Wraps awkwardly on <375px screens.

2

Navbar Takes Space

Sidebar is hidden on mobile (good), but hamburger + top bar buttons ("New", "Running", "History") still consume vertical space. No collapse.

3

Hard-Coded Widths Break

ThinkingBlock: max-w-[380px], dropdowns: w-72 (288px), mobile sidebar: w-[260px] — all overflow on small screens.

4

Tiny Text & Metadata

Timestamps: 9px, tokens: 9px, tool output: 11px, thinking: 11px. Unreadable on mobile.

5

Small Touch Targets

Send button: 32px, tool expand: 28px, copy/edit/retry icons: 12-14px. Apple HIG minimum is 44px.

6

No Safe Area Handling

.safe-bottom class exists in CSS but is never applied. ChatInput can be hidden behind iPhone home indicator.

7

Message Bubbles & Padding

Bubbles at 85% max-width with 16px padding — on 320px screens that's 272px content with only 48px margin. Feels edge-to-edge.
