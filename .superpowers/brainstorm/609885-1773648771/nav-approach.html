Mobile Navigation — No Double Hamburger

Current problem: sidebar hamburger (products) + top bar buttons (session actions) = 2 menus. How to unify?

A

Unified Sidebar (Recommended)

One hamburger opens one sidebar that has everything: products at top, session actions (New, History, Search) below, current session list at bottom. Top bar becomes just: ☰ + session title + ⊕ new.

Pros

  • Single entry point — no confusion
  • Familiar mobile pattern (Slack, Discord, Telegram)
  • Top bar stays ultra-minimal
  • Products + sessions in one place

Cons

  • Sidebar gets denser
  • "New Chat" is 2 taps (hamburger → new) instead of 1
B

Swipe + Minimal Top Bar

Same unified sidebar as A, but also openable by swiping right from left edge (like iOS Mail, Telegram). Top bar: just session title centered, ⊕ on right. No hamburger icon — swipe is the primary trigger.

Pros

  • Maximizes screen space — no hamburger button
  • Feels native on mobile
  • Gesture-driven = modern UX

Cons

  • Not discoverable — new users won't know to swipe
  • Conflicts with browser back-swipe gesture
  • Accessibility concern — gesture-only navigation
C

Hybrid: Hamburger + Swipe

Unified sidebar (like A) with both hamburger button AND swipe-from-edge to open. Best of both — discoverable + fast. This is what Slack, Discord, and Google apps do.

Pros

  • Discoverable (hamburger) + fast (swipe)
  • Industry standard pattern
  • Accessible — button always available

Cons

  • Need to implement touch gesture handling
  • Slightly more code complexity
